提交更改，实际上就是将暂存区的文件提交至当前分支

-m 后面是本次提交的说明，可以输入任意内容。当然最好是有意义的，这样你就可以从历史记录里方便地找到改动记录。

为什么需要 add commit 两步呢？跟git的原理有关，git中有暂存区的概念。你可以add多个文件到暂存区，提交时会一次性将暂存区所有的内容都提交完。

查看提交历史

1>查看当前项目的每次修改记录，包括修改时间，修改的文件或内容

2>如果觉得看的眼花缭乱，可以添加参数–pretty=oneline,它会把修改记录按行显示出来

友情提示：你会看到一长串“bb32347237423c”类似这样的数字，实际上是git的commit id(版本号)。Git的commit id不是1，2，3……递增的数字，而是一个SHA1计算出来的一个非常大的数字，用十六进制表示
