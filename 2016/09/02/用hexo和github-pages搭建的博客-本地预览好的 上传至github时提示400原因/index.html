用hexo和github pages搭建的博客 本地预览ok，上传至github时提示400原因

撰写于 2016-09-02 修改于 2016-09-02 分类 git 标签 git

   用hexo 和github pages搭建的博客，本地预览都是ok的，deploy后 各种访问方式都是报400错误

   刚才解决了这个问题，进入github，找到你的博客文件所在的仓库，点击进去，找到setting 按钮，页面往下拉，看到custom domain,在文本框中填写的域名,比如我的是 www.icidstar.com 然后点击save，搞定，现在在浏览器输入域名访问是ok的，输入http://allida881.github.io 也可以的。

一劳永逸的方法

   上面的方法在调试时是ok，但你每部署一次，都需要重新设置custom domain，这样会比较麻烦。所以推荐的是以下方法：

   在source文件夹下创建CNAME文件，设置你的域名,例如我的为www.icidstar.com，关闭保存。

注意：部署CNAME文件时不能添加任何.txt .md的后缀，可以先创建为.txt文件或其它文件类型，把内容写进去。域名内容设置好以后，开始部署之前一定要将文件的后缀删除掉。不然你在浏览器输入域名，是无法解析的喔。切记！切记！切记！
